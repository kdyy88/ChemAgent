# 总体生化目标
基于骨架跃迁（scaffold hopping）从泽布替尼（zanubrutinib）出发设计 1 个结构相似但骨架不同的新分子，输出可机读的 SMILES/工件并标注与泽布替尼的原子/片段差异点以便下游合成或计算评估。

# 执行管线 (Pipeline)
## 阶段 1：参考分子检索与归档
* **动作意图**: 从公开数据库检索泽布替尼的标准化结构（canonical SMILES、InChI、源 ID）并归档为可供下游工具直接引用的工件。
* **依赖工件 (Inputs)**: 无
* **挂载工具 (Required Tools)**: database-lookup
* **预期产出 (Outputs)**:
  - artifact_id: ref_zanubrutinib (包含 canonical_smiles, InChI, 源数据库 ID 列表，如 PubChem/ChEMBL, 可选 2D SDF)

## 阶段 2：结构标准化与药效团/活性基团注释
* **动作意图**: 使用化学信息学标准化分子（去盐、规范化立体化学/异构体、生成 canonical SMILES），提取 Murcko scaffold，并用 SMARTS 标识并标注可能的反应性活性基团与关键取代基向量（exit vectors）。
* **依赖工件 (Inputs)**: artifact_id: ref_zanubrutinib
* **挂载工具 (Required Tools)**: rdkit
* **预期产出 (Outputs)**:
  - artifact_id: ref_zanubrutinib_cleaned (标准化 SMILES / InChI)
  - artifact_id: ref_murcko_scaffold (Murcko core 为 SMILES)
  - artifact_id: ref_atom_annotation (SMARTS 匹配结果，标注活性位点与取代向量)

## 阶段 3：骨架跃迁设计（生成 1 个候选分子）
* **动作意图**: 在保留泽布替尼关键活性基团（包括用于结合/可共价化学反应的 warhead）和取代基向量的前提下，替换 Murcko core 为一组等排布/等电子替代 core，枚举候选并筛选得出 1 个最终化合物（骨架跃迁产生的新 SMILES）。
* **依赖工件 (Inputs)**: artifact_id: ref_zanubrutinib_cleaned, artifact_id: ref_murcko_scaffold, artifact_id: ref_atom_annotation
* **挂载工具 (Required Tools)**: rdkit
* **预期产出 (Outputs)**:
  - artifact_id: designed_candidate_set (候选 SMILES 列表，含原子映射信息)
  - artifact_id: designed_final_1 (选定的 1 个新分子 SMILES，canonical_smiles)

## 阶段 4：理化与相似性速筛
* **动作意图**: 计算设计分子的分子描述符与指纹，与泽布替尼计算 Tanimoto 相似性，计算关键 ADME/药学指标（分子量、cLogP、TPSA、HBD/HBA、可旋转键），并运行 PAINS/反应性基团过滤器以排除明显的不良基团。
* **依赖工件 (Inputs)**: artifact_id: ref_zanubrutinib_cleaned, artifact_id: designed_final_1
* **挂载工具 (Required Tools)**: rdkit
* **预期产出 (Outputs)**:
  - artifact_id: designed_final_1_props (JSON: 描述符与指纹)
  - artifact_id: similarity_report (包含 Tanimoto 值与关键差异指标)

## 阶段 5：公开库去重/新颖性校验
* **动作意图**: 对最终候选分子执行精确结构查询（exact match）与子结构/相似检索，以识别是否已被收录于主流数据库；若命中，返回匹配条目用于判断是否为已知分子。
* **依赖工件 (Inputs)**: artifact_id: designed_final_1
* **挂载工具 (Required Tools)**: database-lookup
* **预期产出 (Outputs)**:
  - artifact_id: novelty_check (JSON: exact_match_flag, matched_database_ids 列表)

## 阶段 6：差异标注与可视化输出
* **动作意图**: 生成一次对比映射，标出新分子与泽布替尼在原子层面和片段层面的差异；输出可机读的差异清单和带高亮的分子图（或带原子标号的 SMILES/映射）。
* **依赖工件 (Inputs)**: artifact_id: ref_zanubrutinib_cleaned, artifact_id: designed_final_1, artifact_id: ref_atom_annotation, artifact_id: designed_final_1_props
* **挂载工具 (Required Tools)**: rdkit
* **预期产出 (Outputs)**:
  - artifact_id: atom_mapping_diff (映射表: 原子索引对应，差异片段列表)
  - artifact_id: annotated_structures (图像或 SDF 指针，带差异高亮)
  - artifact_id: final_report_machine (JSON: final_smiles, differences_summary, properties, novelty_flag)

# 关键数据缺口
- 是否必须保留泽布替尼的共价/亲电 warhead（例如是否要求保留可与目标半胱氨酸发生共价键的官能团）——此决策会直接影响骨架替换策略；若未指明，计划以“保留 warhead 与取代基向量”为设计默认。 
- 期望的新颖性阈值（例如 Tanimoto 相似性下限或是否允许精确匹配已有数据库条目）；若无阈值，计划将仅报告相似性并标注是否在公开数据库存在精确匹配。
- 是否有明确的合成可及性/合成路径约束（例如允许多步保护/脱保护或仅单步可合成骨架）；若无，设计将优先小范围改动以保留合成可及性。
- 是否需遵守特定 IP/专利回避规则（若需，必须提供专利回避策略或允许的化学空间）。
